CoRe: Context-Robust Remasking for Diffusion Language Models

Kevin Zhai, Sabbir Mollah, Zhenyi Wang Mubarak Shah
University of Central Florida
-
arXiv

This paper introduces Context-Robust Remasking (CoRe), a training-free inference-time method that improves decoding in masked diffusion models (MDMs) by addressing context rigidity.

Abstract

Standard decoding in Masked Diffusion Models (MDMs) is hindered by context rigidity: tokens are retained based on transient high confidence, often ignoring that early predictions lack full context. This creates cascade effects where initial inconsistencies misguide the remaining generation. Existing revision strategies attempt to mitigate this by relying on static confidence scores, but these signals are inherently myopic; inconsistent tokens frequently appear confident to the model itself. To address this, we propose Context-Robust Remasking (CoRe), a training-free framework for inference-time revision. We introduce a new selection paradigm: rather than trusting static token probabilities, we identify context-brittle tokens by probing their sensitivity to adversarial perturbations. We formalize revision as a robust optimization problem targeting worst-case context shifts. CoRe efficiently approximates this objective to expose unstable tokens, prioritizing them for revision. On LLaDA-8B-Base, CoRe delivers consistent improvements across reasoning and code benchmarks, outperforming compute-matched baselines and boosting performance on code generation (MBPP) by up to 9.2%

TL;DR

Problem

Early tokens, generated under partial context, in masked diffusion decoding get “locked in”, causing cascading structural errors.

Idea

Identify context-brittle tokens by stress-testing them under perturbed context, and remask them for revision.

Method

Mask a candidate subset, compute instability scores, and revise only the most unstable tokens.

Outcome

More consistent decoding, especially on structure-sensitive tasks (e.g., code, math reasoning).

Method Overview

CoRe revises tokens based on context brittleness: if a token’s likelihood collapses when its context is perturbed, it’s prioritized for revision.

  1. Pick candidate set of tokens to perturb based on top-2 margin.
  2. Perturb context by masking the candidate set.
  3. Score instability of each token in the candidate set under the perturbed context.
  4. Revise the most unstable tokens using predictions from the perturbed pass.

Candidate Selection

CoRe restricts perturbation to a small candidate set selected via top-2 margin, ensuring that only ambiguous tokens are evaluated while keeping computation minimal.

Context Perturbation

By masking the candidate set, CoRe creates a controlled worst-case context shift that exposes hidden dependencies between tokens and their surrounding context.

Instability Scoring

Tokens are ranked by the drop in likelihood between the original and perturbed passes, providing a dynamic, context-aware signal for revision.

Selective Revision

Only the most unstable tokens are remasked and regenerated, preventing unnecessary disruption of stable parts of the sequence.

Results

Across all benchmarks, CoRe consistently improves performance over both confidence-based and margin-based revision strategies. The gains are especially pronounced on structure-sensitive code generation tasks such as HumanEval and MBPP, where CoRe substantially reduces cascading errors introduced by early, brittle token decisions. These improvements hold across different base selection schemes, demonstrating that robustness-based remasking is complementary to existing confidence heuristics.
